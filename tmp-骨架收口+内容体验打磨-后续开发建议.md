# 骨架收口+内容体验打磨-后续开发建议

## 1. 接入内容

```md
要求：

1. 每个 Markdown 页面都有对应路由；
2. 支持代码块高亮；
3. 支持右侧目录；
4. 支持 frontmatter；
5. 支持标签显示；
6. 支持上一篇 / 下一篇；
7. 支持 Prompt 一键复制；
8. 支持案例筛选；
9. 支持资源卡片；
10. 支持全文搜索。
```

## 2. 给网站首页的内容提示词
生成首页文案时使用：

```md
请为 Codex 从 0 到 1 教程网站生成首页内容。

首页目标：

让用户在 30 秒内理解：

1. 这个网站教什么；
2. 为什么现在要学习 Codex；
3. 学完能获得什么能力；
4. 应该从哪里开始；
5. 有哪些实战案例；
6. 如何把 Codex 用到真实开发、自媒体和知识管理中。

首页模块：

1. Hero 区域；
2. Codex 能力概览；
3. 学习路线；
4. 适合人群；
5. 真实案例；
6. 工具链；
7. 推荐学习路径；
8. FAQ；
9. 开始学习按钮。

风格：

- 现代；
- 简约；
- 技术感；
- 但不要过度营销；
- 面向普通用户和程序员；
- 强调实践。
```

## 4. 给知识管理模块的提示词

```md
请为网站生成“Codex + 知识管理”模块。

请给出具体目录结构、模板和示例。
```

## 补充模块的提示词

我建议再补充下面几个模块。

### 1. Codex 安全边界
原因：AI Coding 很容易误删文件、泄露密钥、执行危险命令。

建议页面：

```txt
/learn/safety
```

内容包括：
- 权限控制；
- 沙箱；
- Git 分支；
- commit 前检查；
- 敏感信息保护；
- .env 文件保护；
- 危险命令确认；
- 生产环境禁止直接操作。

### 2. Codex 项目上下文工程
建议页面：

```txt
/learn/context-engineering
```

内容包括：

- 如何写项目 README；
- 如何写 AGENTS.md；
- 如何写 TASK.md；
- 如何写 SPEC.md；
- 如何写 ARCHITECTURE.md；
- 如何让 Codex 快速理解项目；
- 如何避免每次重新解释。

### 3. Codex 与其他 AI Coding 工具对比
建议页面：

```txt
/resources/comparison
```

内容包括：

- Codex；
- Claude Code；
- Cursor；
- GitHub Copilot；
- Devin；
- Gemini CLI；
- OpenClaw；
- Hermes；
- 本地模型 Coding Agent。


### 4. Codex 日志系统
