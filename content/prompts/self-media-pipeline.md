# 适用场景
- 把研究内容变成多平台发布包。
- 需要标题、摘要、CTA、配图和复用版本。

# 核心观点
- 一份研究可以拆成多个平台版本。
- 事实核对必须和平台改写一起做。

# 可复用知识
- 研究、脚本、发布和复用要分层。
- 不同平台要有不同长度和格式。

# 可复制模板
```text
主题：{{topic}}
平台：{{platforms}}
受众：{{audience}}
核心论点：{{claims}}
事实来源：{{sources}}
输出：标题、摘要、正文、CTA、配图建议、可复用版本
完成标准：{{done_when}}
```

# 最佳实践
- 先统一核心论点。
- 再做平台改写。
- 发布前做事实核对。

# 常见坑
- 没区分平台格式。
- 直接复制研究原文。
- 没有做事实检查。

# 测试与验证方法
- 检查标题和摘要是否适配平台。
- 检查引用是否能回溯。
- 检查复用版本是否偏题。
