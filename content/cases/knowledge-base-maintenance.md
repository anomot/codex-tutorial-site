# 场景
- 需要整理研究笔记、修复链接、补摘要和更新索引。
- 希望知识库能长期保持可检索和可复用。

# 前置条件
- 已有一批研究或 wiki 页面。
- 知道哪些链接或摘要需要维护。
- 有一个固定的标签和目录策略。

# 真实操作流程
1. 让 Codex 扫一遍现有笔记并总结缺口。
2. 补充摘要、链接和标签。
3. 把重复内容合并成更稳定的知识页。
4. 用一次检索测试确认能找回关键内容。

# 核心观点
- 知识管理的重点是可检索，不是堆数量。
- 链接和 checkpoint 是长期记忆的关键。

# 可复用知识
- 每次大任务都要留一个摘要和链接。
- 重复知识应该合并进一个主页。

# 最佳实践
- 保持主页短，细节放到子页。
- 清理过期内容时保留来源。
- 给关键决策写上原因。

# 常见坑
- 只写结论，不留出处。
- 页面之间没有链接。
- 知识库变成无法检索的堆栈。

# 测试与验证方法
- 随机抽一个问题，看能否从知识库直接找到答案。
- 检查链接是否可双向回跳。
- 确认摘要不会丢失关键信息。
