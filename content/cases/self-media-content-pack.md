# 场景
- 把一份研究或教程变成可发布的自媒体内容。
- 要同时产出标题、摘要、CTA、配图建议和复用版本。

# 前置条件
- 已经有研究材料或主稿。
- 知道目标平台和受众。
- 能做事实核对和长度控制。

# 真实操作流程
1. 先整理研究结论和核心论点。
2. 分别生成博客、X 帖子、newsletter 或短视频脚本版本。
3. 核对事实、链接和措辞。
4. 打包成发布包并保留复用模板。

# 核心观点
- 自媒体工作流的关键是把一份材料拆成多个渠道版本。
- 事实核对和平台适配必须一起做。

# 可复用知识
- 研究、脚本、发布和复用要分层。
- 同一主题可以产出多种长度和风格。

# 最佳实践
- 先统一主张，再分平台改写。
- 发布前先做事实检查。
- 保留可复用模板和 CTA。

# 常见坑
- 直接复制研究结论发布，没有重写。
- 不同平台的格式和长度没区分。
- 缺少事实校验。

# 测试与验证方法
- 检查标题和摘要是否适合目标平台。
- 确认引用和链接都能回溯。
- 确认复用版本没有歪曲原意。

# 下一步学习建议
- 看 self-media lesson。
- 再看 image/video 工作流。
- 然后做 self-media prompt 模板。
