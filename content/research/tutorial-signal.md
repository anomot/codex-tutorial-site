# 核心观点
- 新手教程最有用的不是“功能清单”，而是“第一次成功”的路径：安装、登录、连仓库、发任务、看 diff、验结果。
- 视频内容更适合展示端到端流程，博客更适合拆解 prompt 模式、AGENTS.md、sandbox 和验证习惯。
- 教程如果只讲“怎么用”，读者很快会忘；如果讲“怎么验证”，读者更容易迁移到自己的项目。

# 可复用知识
- 先演示一条最短闭环，再扩展到重构、review、MCP、skills。
- 每个教程段落都要带一个可复制 prompt。
- 讲解时把“成功标准”和“失败时怎么排查”放在同一小节。

# 最佳实践
- YouTube 负责“看见流程”，文字教程负责“复述步骤”。
- 结合仓库规则文件、测试命令和 review 习惯，做成模板化教程。
- 用真实 repo 示例解释上下文注入与验证过程。

# 常见坑
- 把教程写成产品介绍页，缺少实际操作步骤。
- 只展示一次成功结果，不展示验证和排错。
- 忽略用户一开始最关心的“我装完后下一步是什么”。

# 适合加入网站的位置

- “第一个任务”示例页。
