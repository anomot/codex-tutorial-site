# 核心观点
- AI Coding 需要知识管理，因为模型每次都会丢失上下文。
- 长期可复用的知识，应该沉淀成本地文档、错误库和 Prompt 库。
- 知识管理系统的核心，不是存得多，而是找得快、接得上、能复用。

# 可复用知识
- Research、Wiki、Lessons、Cases、Prompts、Logs 是一套稳定的知识结构。
- Obsidian 或本地 Markdown 很适合做 Codex 的长期知识底座。
- MOC 能把分散文档串成可导航的知识网络。

# 最佳实践
- 每次任务后都写复盘。
- 每周清理重复条目和失效条目。
- 为常见错误建立错误库。
- 为常见任务建立 Prompt 库和案例库。

# 常见坑
- 只记录结果，不记录过程。
- 文档没有链接，后期找不到。
- 没有把错误和复盘沉淀成可复用知识。

# Prompt 示例
```text
请把这次 Codex 任务整理成知识记录：
- 任务目标
- 输入材料
- 关键决策
- 验证方式
- 输出结果
- 复盘
- 可复用经验
```
