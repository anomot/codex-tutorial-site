# 核心观点
- 错误库是把失败变成资产的关键机制。
- 它应该和 Prompt 库、案例库、日志系统互相连接。
- 错误库越稳定，后续 Codex 使用越省时间。

# 可复用知识
- 错误现象、根因、修复、验证、预防五项最重要。
- 用标签和链接让错误条目可检索。
- 错误库适合做成“短条目 + 关联链接”的形式。

# 最佳实践
- 每条错误只讲一个问题。
- 记录可验证的修复方式。
- 定期去重和合并。

# 常见坑
- 错误描述过长。
- 没有上下文链接。
- 只写“已解决”，不写怎么解决。

# Prompt 示例
```text
请把这次错误整理成错误库条目：
1. 错误现象；
2. 根因；
3. 修复；
4. 验证；
5. 预防。
```
