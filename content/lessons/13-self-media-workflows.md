# 核心观点
- 自媒体流程的关键是把选题、脚本、发布和复用连起来。
- Codex 更适合做研究、整理和标准化检查。
- 教程应该把内容生产做成流水线。

# 适用场景
- X 帖子、博客、newsletter、短视频脚本、教程复用。
- 需要快速把研究结果变成发布包。
- 需要统一标题、摘要、配图和 CTA。

# 可复用知识
- 选题、事实、结构、发布检查要分开。
- 一份研究可以产出多种发布形态。
- 复用模板能降低内容生产成本。

# 步骤
1. 确定主题和受众。
2. 把研究整理成提纲和论点。
3. 生成不同平台的发布版本。
4. 做事实校验和最后的发布检查。

# 最佳实践
- 先建立统一的内容模板。
- 对外发布前先做事实核对。
- 把一篇内容拆成多种平台版本。

# 常见坑
- 只生成文案，不做事实检查。
- 不同平台的长度和格式没区分。
- 没有把内容做成可复用生产线。

# 测试与验证方法
- 检查标题、摘要、CTA 是否匹配平台。
- 确认事实和引用都能追溯。
- 确认复用版本不会偏题。
