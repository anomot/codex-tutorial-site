# 核心观点
- 知识管理不是堆笔记，而是能被检索和复用的上下文。
- 上下文持久性来自索引、标签、链接和 checkpoint。
- 好知识库能让 Codex 和人都更快接上前文。

# 适用场景
- 项目复盘和决策记录。
- 建立长期可复用的任务库和 prompt 库。
- 整理研究笔记并保持可追溯。

# 可复用知识
- 每次大任务都写一个 checkpoint。
- 链接比重复描述更有价值。
- 摘要要短，原始来源要可回跳。

# 步骤
1. 记录当前目标和上下文。
2. 把决策、证据和链接写入固定位置。
3. 给条目添加标签和相关页链接。
4. 定期清理过时内容并保留历史线索。

# 最佳实践
- 把“做过什么”与“为什么这样做”分开记录。
- 为每个主题保留一个可检索的主页。
- 让教程里所有关键结论都有来源。

# 常见坑
- 只记结论，不记上下文和出处。
- 笔记没有标签和链接，后面找不到。
- 复盘写得太长，没人会再看。

# 测试与验证方法
- 过几天再回看，能否快速找回关键决策。
- 让 Codex 只根据知识库回答一次问题，检查是否能复述正确。
- 检查新旧页面之间是否互相可达。

# 下一步学习建议
- 用一个真实项目做知识页。
- 把一个 prompt 库接到案例页。
- 再回到扩展与连接器页。

# 可复制 Prompt
```text
请把这次会话整理成一条知识记录：
目标、关键决策、证据链接、可复用结论、待验证点、下一步。
```
