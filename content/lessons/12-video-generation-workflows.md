# 核心观点
- 视频流程比图片更需要脚本、分镜和审核。
- Codex 更适合做结构化策划和验证清单。
- 教程要把视频生产拆成可检查的阶段。

# 适用场景
- 教程视频、产品演示、短视频、课程片段。
- 需要把一段主题拆成脚本和镜头。
- 需要统一脚本、素材和发布检查。

# 可复用知识
- 先写脚本，再做镜头，再做审核。
- 字幕、节奏和音画同步都要验证。
- 每个阶段都要有退出条件。

# 步骤
1. 写视频目标、受众和时长。
2. 生成脚本和 shot list。
3. 准备素材并检查镜头顺序。
4. 导出后检查字幕、节奏和同步。

# 最佳实践
- 每个视频都保留一个脚本母版。
- 先做小片段验证，再扩到完整视频。
- 把封面、标题和简介也纳入检查。

# 常见坑
- 脚本太长，镜头无法落地。
- 素材风格不统一。
- 忽略字幕和节奏的最终审核。

# 测试与验证方法
- 看预览是否满足时长和节奏。
- 检查字幕、旁白和镜头切换是否同步。
- 确认导出格式适合发布平台。
